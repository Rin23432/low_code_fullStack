# 面试讲解包（最小版）

## 5 个讲解点
1. 架构拆分：编辑端、消费端、Mock 服务解耦。
2. 技术取舍：React + Next.js + Koa 的学习曲线与收益。
3. 性能思路：BFF 减少前端直连复杂度，便于聚合与缓存。
4. 工程化：文档先行、脚本化启动、契约优先。
5. 可扩展：后续可替换 Mock 为真实后端并保持前端接口稳定。

## 3 个现场演示脚本（2-3 分钟）
1. 启动演示：三服务启动 -> 首页访问 -> 一次请求成功。
2. 链路演示：页面请求 BFF，再转发到 Mock，展示响应结构。
3. 变更演示：在 API 契约文档新增字段并说明如何同步实现。

## 5 个高频问答
- Q: 为什么要 BFF？
- A: 统一前端数据形态，降低客户端复杂度并便于演进。

- Q: 为什么保留 Mock？
- A: 前后端可并行开发，减少联调等待。

- Q: 如何保证多人协作不混乱？
- A: 先契约后实现，配合 PR 检查与提交规范。

- Q: 如何做性能优化？
- A: 从请求链路、渲染路径、资源体积三个方向分层优化。

- Q: 后续怎么上生产？
- A: 先接入 CI 与回归，再替换 Mock 并增加监控。
